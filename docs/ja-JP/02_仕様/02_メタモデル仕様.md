# メタモデル仕様

## 1. 目的

本仕様は、SansaVRM Core において扱う内部メタモデルの構造を定義する。

メタモデルは SansaVRM Format の論理基盤であり、
すべてのモデル・モジュール・状態・権利・収益情報は本モデルに基づいて表現される。

---

## 2. 基本方針

- フォーマット非依存とする
- Slot ベースで統一する
- 構造・状態・権利・収益を分離する
- すべての要素は ID により識別する
- 非可逆情報は diagnostics に保持する

---

## 3. 全体構造

メタモデルは以下の3層で構成される。

- Core Layer（論理構造）
- Slot Layer（接続・機能単位）
- Extension Layer（フォーマット依存情報）

---

## 4. Core Layer

### 4.1 Model

モデルは最上位の構造単位である。

#### 属性

- model_id
- modules[]
- slots[]
- states[]
- rights
- revenue
- diagnostics
- connections[]

connections はモデル全体の接続関係を表す。

- from_slot_id
- to_slot_id
- connection_type

---

### 4.2 Module

モジュールは構造および機能の最小単位である。

#### 属性

- module_id
- type
- slots[]
- properties
- state_refs[]

---

### 4.3 Property

Property は、Module または Slot に付与される汎用属性定義である。

#### 属性

- property_id
- key
- value
- value_type
- constraints
- metadata

#### value_type

Property の値型は以下を基本とする。

- string
- number
- boolean
- object
- array

#### constraints

Property に対する制約は以下を許容する。

- required
- enum
- min
- max
- min_length
- max_length
- pattern

#### metadata

Property の補助情報として以下を保持可能とする。

- description
- unit
- source
- notes

---

## 5. Slot Layer

Slot はすべての接続・制御の基盤となる。

### 5.1 Slot 共通属性

すべての Slot は以下の共通属性を持つ。

- slot_id
- slot_type
- owner_module_id
- target_slot_types[]
- current_connections[]
- connection_rules
- constraints
- properties

constraints は Property と同様の構造を持つ条件定義とする。

#### target_slot_types

接続可能な相手 Slot 種別を定義する。

#### current_connections

現在接続中の相手参照を保持する。

#### connection_rules

接続規則として以下を保持可能とする。

- min_connections
- max_connections
- exclusive
- replace_mode
- priority

#### constraints

接続時に満たすべき条件を保持する。

#### properties

Slot 固有の追加属性を Property として保持する。

---

### 5.2 Slot種別

#### Structure Slot
- モジュール接続

#### Equipment Slot
- 装着・取り外し

#### State Slot
- 状態管理

#### Rights Slot
- 権利情報

#### Revenue Slot
- 収益情報

---

## 6. Extension Layer

Extension Layer は、SansaVRM Core の論理構造とは独立して、
外部フォーマット固有情報を保持する領域である。

Extension Layer は SansaVRM Format における拡張領域に直接対応する。

### 6.1 目的

- フォーマット固有情報の保持
- 非可逆変換情報の補完
- 外部フォーマットとの往復変換支援

### 6.2 属性

- format_type
- extensions
- source_raw
- diagnostics

### 6.3 format_type

format_type は以下を基本とする。

- glTF
- VRM
- URDF
- custom

### 6.4 extensions

フォーマット固有の拡張データを保持する。

### 6.5 source_raw

元フォーマットの未正規化情報を保持する。

### 6.6 diagnostics

拡張層で発生した変換上の問題を保持する。

---

## 7. State

### 7.1 State 定義

状態は識別子により管理される。

#### 属性

- state_id
- category
- conditions
- actions
- priority
- enabled

#### category

状態カテゴリは以下を基本とする。

- Expression
- Configuration
- Equipment
- Visibility

#### conditions

conditions は状態適用条件を表す条件式であり、以下を持つ。

- operator（AND / OR / NOT）
- expressions[]

expressions[] の各要素は以下を持つ。

- left
- comparator
- right

comparator は以下を基本とする。

- eq
- neq
- gt
- gte
- lt
- lte
- in
- not_in

#### actions

状態適用時の動作を保持する。

#### priority

複数状態が競合する場合の優先順位を保持する。

#### enabled

当該状態定義が有効かどうかを示す。

---

### 7.2 State Action

### 7.2 State Action

状態変更時の動作は以下を基本とする。

- module_enable(module_id)
- module_disable(module_id)
- slot_bind(slot_id, target_slot_id)
- slot_unbind(slot_id, target_slot_id)
- expression_change(expression_id)
- property_override(property_id, value)
- visibility_change(target_id, visible)

## 10. Compatibility

Compatibility は、構成要素間の適合条件を定義する。

### 10.1 判定対象

- structure
- slot
- tag
- constraint

### 10.2 判定入力

Compatibility 判定は以下の情報を入力として扱う。

- required_slots[]
- forbidden_slots[]
- required_tags[]
- forbidden_tags[]
- constraint_rules[]
- target_module_types[]

### 10.3 constraint_rules

constraint_rules は個別制約を表し、以下を持つ。

- rule_id
- condition
- result
- severity

### 10.4 判定結果

- ALLOW
- CONDITIONAL
- DENY

---

## 11. diagnostics

すべての非正規情報を保持する。

### 11.1 対象

- 非可逆変換
- 判定失敗
- 権利違反
- 構造不整合

---

## 12. ID体系

### 12.1 要件

- 一意であること
- グローバル / ローカル区別可能
- 参照可能であること

### 12.2 ID形式

IDは以下の形式を基本とする：

- UUID（推奨）
- または パス形式（例: model/module/slot）

### 12.3 スコープ

- model_id：グローバル
- module_id：model内一意
- slot_id：module内一意

### 12.4 参照方式

参照は以下で行う：

- ID直接参照
- パス参照（例: module_id.slot_id）

---

## 13. シリアライズ方針

メタモデルは SansaVRM Format としてシリアライズされる。

- JSON構造を基本とする
- glTF extensions にマッピング可能とする
- 部分シリアライズを許容する

---

## 14. 非可逆変換

変換時に失われる情報は以下に保持する。

- diagnostics
- source_raw
