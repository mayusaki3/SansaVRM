# VRM 0.x import詳細設計

## 1. 目的

VRM 0.x ファイルを SansaVRM Model に変換する import 処理を設計する。

対象は `crates/sansavrm-vrm` とし、VRM 0.x 固有構造を core に持ち込まない。

---

## 2. 入力対象

VRM 0.x は glTF 2.0 をベースとし、VRM固有情報を `extensions.VRM` に保持する。

import 対象は以下とする。

| 入力要素 | VRM 0.x上の位置 | SansaVRM変換先 |
|---|---|---|
| node | `nodes` | Module |
| hierarchy | `nodes[].children` | Connection |
| mesh | `meshes` | Geometry |
| humanoid | `extensions.VRM.humanoid` | PropertyType::Rig + Binding |
| blendshape | `extensions.VRM.blendShapeMaster` | PropertyType::Expression |
| material | `extensions.VRM.materialProperties` / `materials` | PropertyType::Material |
| texture | `textures` / `images` / `samplers` | PropertyType::Texture |
| meta | `extensions.VRM.meta` | PropertyType::Metadata |

---

## 3. import基本方針

### 3.1 変換責務

VRM 0.x 固有の読み取り、解釈、互換変換は `crates/sansavrm-vrm` が担当する。

SansaVRM core には以下のみを渡す。

- Module
- Connection
- Geometry
- Property
- Binding
- passthrough情報

---

## 4. 処理順序

import は以下の順で実行する。

1. glTF/GLB 読み込み
2. VRM 0.x extension 検出
3. node → Module 変換
4. node hierarchy → Connection 変換
5. mesh → Geometry 変換
6. humanoid → Rig Property + Binding 変換
7. blendShapeMaster → Expression Property 変換
8. materialProperties / materials → Material Property 変換
9. textures / images / samplers → Texture Property 変換
10. meta → Metadata Property 変換
11. extensions / extras / unknown fields → passthrough 保持
12. validator 実行

---

## 5. node → Module 変換

### 5.1 入力

- `nodes[]`
- node index
- node name
- transform
  - translation
  - rotation
  - scale
  - matrix

### 5.2 出力

各 node を Module に変換する。

### 5.3 要件

- node index と Module ID の対応表を作成する
- humanoid / mesh / hierarchy 変換で同じ対応表を使用する
- node name は保持する
- transform は Module 側の属性として保持する
- extensions / extras は node単位 passthrough として保持する

---

## 6. hierarchy → Connection 変換

### 6.1 入力

- `nodes[].children`

### 6.2 出力

親子関係を Connection として保持する。

### 6.3 要件

- parent node index → child node index を Module ID に変換する
- 循環参照がある場合はエラー
- root node は親なしとして扱う
- 複数rootは許容する

---

## 7. mesh → Geometry 変換

### 7.1 入力

- `meshes[]`
- `nodes[].mesh`

### 7.2 出力

mesh を Geometry として保持する。

### 7.3 要件

- mesh index と Geometry ID の対応表を作成する
- Module が参照する mesh index を Geometry ID に変換する
- primitive / attributes / indices / morph target は保持する
- 未対応の mesh 情報は Geometry 拡張データまたは passthrough に保持する

---

## 8. humanoid → Rig Property + Binding 変換

### 8.1 入力

- `extensions.VRM.humanoid.humanBones`

### 8.2 出力

- PropertyType::Rig
- PropertyContext::Simulation
- PropertyContext::Execution
- Binding

### 8.3 要件

- human bone 名を正規化する
- bone に紐づく node index を Module ID に変換する
- 1 bone → 1 Module とする
- node index 未解決はエラー
- 重複boneはエラー
- optional bone 欠落は許容する

---

## 9. blendShapeMaster → Expression Property 変換

### 9.1 入力

- `extensions.VRM.blendShapeMaster.blendShapeGroups`

### 9.2 出力

- PropertyType::Expression
- PropertyContext::Rendering
- PropertyContext::Execution

### 9.3 要件

- blendShapeGroupごとに Expression Property を生成する
- presetName / name を保持する
- binds / materialValues を保持する
- mesh index / morph target index の参照を Geometry ID に変換する
- 未対応フィールドは Property 拡張データへ保持する

---

## 10. material → Material Property 変換

### 10.1 入力

- `materials[]`
- `extensions.VRM.materialProperties[]`

### 10.2 出力

- PropertyType::Material
- PropertyContext::Rendering
- PropertyContext::Conversion

### 10.3 要件

- glTF material と VRM materialProperties の対応を保持する
- shader名を保持する
- floatProperties / vectorProperties / textureProperties / keywordMap / tagMap を保持する
- MToon 固有情報は Material Property 拡張データとして保持する
- 未対応フィールドは material単位 passthrough に保持する

---

## 11. texture → Texture Property 変換

### 11.1 入力

- `textures[]`
- `images[]`
- `samplers[]`

### 11.2 出力

- PropertyType::Texture
- PropertyContext::Rendering
- PropertyContext::Conversion

### 11.3 要件

- texture index と Texture Property ID の対応表を作成する
- image / sampler 参照を保持する
- URI / bufferView / mimeType を保持する
- Material Property から Texture Property を参照できるようにする

---

## 12. meta → Metadata Property 変換

### 12.1 入力

- `extensions.VRM.meta`

### 12.2 出力

- PropertyType::Metadata
- PropertyContext::Description

### 12.3 要件

- title / version / author / contactInformation / reference を保持する
- allowedUserName / violentUssageName / sexualUssageName / commercialUssageName を保持する
- licenseName / otherLicenseUrl を保持する
- texture参照がある場合は Texture Property ID に変換する
- 未対応フィールドは modelルート passthrough に保持する

---

## 13. passthrough保持

### 13.1 対象

以下は破棄せず保持する。

- 未対応 extension
- extras
- unknown fields
- 将来exportに必要となる可能性があるVRM固有情報

### 13.2 スコープ

- node単位
- material単位
- mesh / geometry単位
- modelルート単位

### 13.3 export時

元の配置階層へ復元可能な形式で保持する。

---

## 14. validator連携

import 完了後、SansaVRM validator を実行する。

### 14.1 期待される分類

| PropertyType | PropertyContext |
|---|---|
| Rig | Simulation / Execution |
| Expression | Rendering / Execution |
| Material | Rendering / Conversion |
| Texture | Rendering / Conversion |
| Metadata | Description |
| Geometry | Rendering / Simulation |

### 14.2 禁止

- PropertyContextを拡張しない
- validator分類ルールを変更しない
- VRM 0.x 専用型を core に追加しない

---

## 15. エラーハンドリング

### 15.1 エラー

以下は import 失敗とする。

- GLB / glTF として読めない
- `extensions.VRM` が存在しない
- humanoid必須boneの node index が未解決
- node hierarchy に循環がある
- 重複boneが存在する

### 15.2 警告

以下は import 継続可能とする。

- optional bone 欠落
- 不明bone名
- 未対応 extension
- 未対応 material shader
- 未使用 texture
- meta の一部欠落
