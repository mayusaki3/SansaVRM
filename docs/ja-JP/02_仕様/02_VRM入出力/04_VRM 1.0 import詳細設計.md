# VRM 1.0 import詳細設計

## 1. 目的

VRM 1.0 ファイルを SansaVRM Model に変換する import 処理を設計する。

対象は `crates/sansavrm-vrm` とし、VRM 1.0 固有構造を core に持ち込まない。

---

## 2. 入力対象

VRM 1.0 は glTF 2.0 をベースとし、VRM固有情報を機能別 extension として保持する。

VRM 1.0 では、humanoid / meta / expressions などを含む `VRMC_vrm-1.0`、MToon material の `VRMC_materials_mtoon-1.0`、spring bone の `VRMC_springBone-1.0`、node constraint の `VRMC_node_constraint-1.0` などに分割される。

| 入力要素 | VRM 1.0上の位置 | SansaVRM変換先 |
|---|---|---|
| node | `nodes` | Module |
| hierarchy | `nodes[].children` | Connection |
| mesh | `meshes` | Geometry |
| humanoid | `extensions.VRMC_vrm.humanoid` | PropertyType::Rig + Binding |
| expression | `extensions.VRMC_vrm.expressions` | PropertyType::Expression |
| material | `materials` / `extensions.VRMC_materials_mtoon` | PropertyType::Material |
| texture | `textures` / `images` / `samplers` | PropertyType::Texture |
| meta | `extensions.VRMC_vrm.meta` | PropertyType::Metadata |
| lookAt | `extensions.VRMC_vrm.lookAt` | PropertyType::Expression または passthrough |
| firstPerson | `extensions.VRMC_vrm.firstPerson` | passthrough |
| springBone | `extensions.VRMC_springBone` | Phase 2以降 |
| node constraint | `extensions.VRMC_node_constraint` | Phase 2以降 |

---

## 3. import基本方針

### 3.1 変換責務

VRM 1.0 固有の読み取り、解釈、互換変換は `crates/sansavrm-vrm` が担当する。

SansaVRM core には以下のみを渡す。

- Module
- Connection
- Geometry
- Property
- Binding
- passthrough情報

---

## 4. 処理順序

import は以下の順で実行する。

1. glTF/GLB 読み込み
2. `VRMC_vrm` extension 検出
3. VRM 1.0 extension 群の検出
4. node → Module 変換
5. node hierarchy → Connection 変換
6. mesh → Geometry 変換
7. `VRMC_vrm.humanoid` → Rig Property + Binding 変換
8. `VRMC_vrm.expressions` → Expression Property 変換
9. `materials` / `VRMC_materials_mtoon` → Material Property 変換
10. textures / images / samplers → Texture Property 変換
11. `VRMC_vrm.meta` → Metadata Property 変換
12. lookAt / firstPerson → Property または passthrough 保持
13. springBone / node constraint → passthrough 保持
14. extensions / extras / unknown fields → passthrough 保持
15. validator 実行

---

## 5. node → Module 変換

### 5.1 入力

- `nodes[]`
- node index
- node name
- transform
  - translation
  - rotation
  - scale
  - matrix

### 5.2 出力

各 node を Module に変換する。

### 5.3 要件

- node index と Module ID の対応表を作成する
- humanoid / mesh / hierarchy / expression / constraint 変換で同じ対応表を使用する
- node name は保持する
- transform は Module 側の属性として保持する
- extensions / extras は node単位 passthrough として保持する

---

## 6. hierarchy → Connection 変換

### 6.1 入力

- `nodes[].children`

### 6.2 出力

親子関係を Connection として保持する。

### 6.3 要件

- parent node index → child node index を Module ID に変換する
- 循環参照がある場合はエラー
- root node は親なしとして扱う
- 複数rootは許容する

---

## 7. mesh → Geometry 変換

### 7.1 入力

- `meshes[]`
- `nodes[].mesh`

### 7.2 出力

mesh を Geometry として保持する。

### 7.3 要件

- mesh index と Geometry ID の対応表を作成する
- Module が参照する mesh index を Geometry ID に変換する
- primitive / attributes / indices / morph target は保持する
- 未対応の mesh 情報は Geometry 拡張データまたは passthrough に保持する

---

## 8. humanoid → Rig Property + Binding 変換

### 8.1 入力

- `extensions.VRMC_vrm.humanoid.humanBones`

### 8.2 出力

- PropertyType::Rig
- PropertyContext::Simulation
- PropertyContext::Execution
- Binding

### 8.3 要件

- human bone 名を正規化する
- bone に紐づく node index を Module ID に変換する
- 1 bone → 1 Module とする
- node index 未解決はエラー
- 重複boneはエラー
- optional bone 欠落は許容する
- VRM 1.0 の必須bone条件に従って警告またはエラーを出す

---

## 9. expressions → Expression Property 変換

### 9.1 入力

- `extensions.VRMC_vrm.expressions.preset`
- `extensions.VRMC_vrm.expressions.custom`

### 9.2 出力

- PropertyType::Expression
- PropertyContext::Rendering
- PropertyContext::Execution

### 9.3 要件

- preset expression と custom expression を区別して保持する
- expression 名を保持する
- morphTargetBinds を Geometry ID / morph target index に変換する
- materialColorBinds を Material Property ID に変換する
- textureTransformBinds を Texture Property ID または Material Property ID に関連付ける
- isBinary / overrideBlink / overrideLookAt / overrideMouth を保持する
- 未対応フィールドは Property 拡張データへ保持する

---

## 10. material → Material Property 変換

### 10.1 入力

- `materials[]`
- `materials[].extensions.VRMC_materials_mtoon`

### 10.2 出力

- PropertyType::Material
- PropertyContext::Rendering
- PropertyContext::Conversion

### 10.3 要件

- glTF material を Material Property として保持する
- `VRMC_materials_mtoon` が存在する場合、MToon 固有情報を Material Property 拡張データとして保持する
- pbrMetallicRoughness 等の標準 glTF material 情報も保持する
- textureInfo は Texture Property ID に変換する
- 未対応フィールドは material単位 passthrough に保持する

---

## 11. texture → Texture Property 変換

### 11.1 入力

- `textures[]`
- `images[]`
- `samplers[]`

### 11.2 出力

- PropertyType::Texture
- PropertyContext::Rendering
- PropertyContext::Conversion

### 11.3 要件

- texture index と Texture Property ID の対応表を作成する
- image / sampler 参照を保持する
- URI / bufferView / mimeType を保持する
- Material Property / Expression Property から Texture Property を参照できるようにする

---

## 12. meta → Metadata Property 変換

### 12.1 入力

- `extensions.VRMC_vrm.meta`

### 12.2 出力

- PropertyType::Metadata
- PropertyContext::Description

### 12.3 要件

- name / version / authors / copyrightInformation / contactInformation / references を保持する
- thirdPartyLicenses / thumbnailImage を保持する
- licenseUrl / avatarPermission / commercialUsage / creditNotation 等を保持する
- thumbnailImage 参照がある場合は Texture Property ID に変換する
- 未対応フィールドは modelルート passthrough に保持する

---

## 13. lookAt / firstPerson の扱い

### 13.1 lookAt

初期実装では passthrough 保持を基本とする。

ただし、expression との関係が明確に解釈可能な場合は Expression Property 拡張データとして保持してよい。

### 13.2 firstPerson

初期実装では modelルート passthrough として保持する。

---

## 14. springBone / node constraint の扱い

### 14.1 springBone

Phase 1 では変換対象外とし、`VRMC_springBone` を modelルート passthrough として保持する。

Phase 2以降で Physics または Constraint Property への変換を検討する。

### 14.2 node constraint

Phase 1 では変換対象外とし、`VRMC_node_constraint` を node単位または modelルート passthrough として保持する。

Phase 2以降で Constraint Property への変換を検討する。

---

## 15. passthrough保持

### 15.1 対象

以下は破棄せず保持する。

- 未対応 extension
- extras
- unknown fields
- VRM 1.0 extension のうち Phase 1 で直接変換しないもの
- 将来exportに必要となる可能性があるVRM固有情報

### 15.2 スコープ

- node単位
- material単位
- mesh / geometry単位
- expression単位
- modelルート単位

### 15.3 export時

元の配置階層へ復元可能な形式で保持する。

---

## 16. validator連携

import 完了後、SansaVRM validator を実行する。

### 16.1 期待される分類

| PropertyType | PropertyContext |
|---|---|
| Rig | Simulation / Execution |
| Expression | Rendering / Execution |
| Material | Rendering / Conversion |
| Texture | Rendering / Conversion |
| Metadata | Description |
| Geometry | Rendering / Simulation |

### 16.2 禁止

- PropertyContextを拡張しない
- validator分類ルールを変更しない
- VRM 1.0 専用型を core に追加しない

---

## 17. エラーハンドリング

### 17.1 エラー

以下は import 失敗とする。

- GLB / glTF として読めない
- `VRMC_vrm` extension が存在しない
- humanoid必須boneの node index が未解決
- node hierarchy に循環がある
- 重複boneが存在する
- expression の参照先 mesh / material が未解決

### 17.2 警告

以下は import 継続可能とする。

- optional bone 欠落
- 不明bone名
- 未対応 extension
- 未対応 material extension
- 未使用 texture
- meta の一部欠落
- springBone / node constraint の直接変換未対応
