# 物理・制御メタモデル仕様

## 1. 目的

本仕様は、SansaVRM における物理・制御・センサ・アクチュエータ情報の表現方法を定義する。

本仕様の目的は以下とする。

- 物理・制御に関する責務の分離
- Connection / Slot / Property の役割の明確化
- 外部シミュレータ（例：MuJoCo）との連携基盤の確立
- Validator による整合性検証の前提定義

---

## 2. 基本方針

- 接続関係は `Connection` により一元管理する
- `Slot` は接続の補助構造として扱う
- 物理・制御・センサ・アクチュエータ情報は `Property` により表現する
- 構造（接続）と意味（物理・制御）を分離する
- 外部エンジン依存の情報は最小限に留める

---

## 3. Slot の役割

Slot は補助構造として以下を保持する。

- 接続制約

  - `target_slot_types`
  - `max_connections`
  - `exclusive`

- インターフェース定義

  - 接続可能な型の定義
  - 入出力の区別（input / output）

- 役割分類

  - 例：physics / control / sensor / actuator

Slot は以下を保持しない：

- 接続関係そのもの
- 実体の物理値

Slot は Module に属する構造とする。

各 Slot は1つの Module に紐付く。

---

## 4. Connection の役割

Connection は接続の唯一の実体とする。

保持する情報：

- `connection_id`
- `from_id`
- `to_id`
- `connection_type`
- `enabled`
- `conditions`

役割：

- Module 間の接続関係を表現する
- 接続の方向性を持つ（from → to）
- joint / attach / signal などの意味を持つ

Connection は Model に属するグローバル構造とする。

---

## 5. Property の役割

Property は値および属性情報の格納に使用する。

保持対象：

- 物理量
- 制御パラメータ
- センサ値
- アクチュエータ設定

特徴：

- 型を持つ（string / number / boolean / array 等）
- Validator により型整合性が検証される
- Slot や Module に紐付け可能

Property は Module または Slot に紐付く。

- Module：構造全体に関する属性
- Slot：接続・制約・インターフェースに関する属性

---

## 6. Physics 定義

物理情報は Property により表現する。

対象例：

- 質量（mass）
- 慣性（inertia）
- 摩擦（friction）
- 衝突形状（collision）

方針：

- 構造（接続）とは分離する
- エンジン依存表現は持たない

---

## 7. Control 定義

制御情報は Property により表現する。

対象例：

- 制御ゲイン（P / I / D）
- 目標値（target）
- 制御モード

方針：

- Connection を通じて制御信号が伝達される
- 制御ロジック自体は本仕様のスコープ外

---

## 8. Sensor 定義

センサ情報は Property により表現する。

対象例：

- 位置
- 速度
- 力
- 接触情報

特徴：

- 読み取り専用の値として扱う
- State と連携可能

---

## 9. Actuator 定義

アクチュエータ情報は Property により表現する。

対象例：

- トルク
- 力
- 速度制御値

特徴：

- Control と連携して動作する
- 出力側のインターフェースとして扱う

---

## 10. State 連携

State は動的状態を管理する。

連携対象：

- Property の現在値
- センサ値
- 制御状態

特徴：

- 静的定義（Property）と動的値（State）を分離する

---

## 11. Validator 連携

Validator は以下を検証する。

- Property の型整合性
- Slot の接続制約
- Connection の整合性
- State の参照整合性

エラーは diagnostics として出力される。

---

本仕様は以下と連携する。

---

## 13. 非スコープ

本仕様の対象外：

- 物理エンジンの実装
- 制御アルゴリズム
- シミュレーション実行
- UI / 可視化
- ネットワーク同期
