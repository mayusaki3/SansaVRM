# 仕様概要

## 1. 目的

本仕様は、SansaVRM に関する全体仕様の構造、役割、および相互関係を定義する。

本仕様は以下を目的とする。

- 各仕様ドキュメントの位置付けを明確にする
- SansaVRM の全体アーキテクチャを統一的に理解可能とする
- 実装・テストの前提となる設計境界を定義する
- Format / Core / Runtime の責務を明確に分離する

---

## 2. SansaVRM の構成

SansaVRM は以下の3層構造で構成される。

### 2.1 SansaVRM Format

- 外部交換フォーマット
- glTF 拡張として表現
- JSON Schema により検証可能

対応仕様：

- [glTF拡張仕様](./03_glTF拡張仕様.md)
- [JSONスキーマ仕様](./04_JSONスキーマ仕様.md)

---

### 2.2 SansaVRM Core

- 内部データモデル
- モデル操作・状態管理・接続制御を担当
- Validator による整合性保証

対応仕様：

- [メタモデル仕様](./02_メタモデル仕様.md)
- [Validator実装仕様](./05_Validator実装仕様.md)
- [CoreAPI仕様](06_CoreAPI仕様.md)

---

### 2.3 SansaVRM Runtime

- 実行環境（本仕様の対象外）
- 状態適用・レンダリング・物理処理などを担当
- Core により生成された状態を実行する環境
- Core API の結果を適用する責務を持つ
- 本仕様では Runtime の動作仕様は定義しないが、
  Core により生成される状態・接続・プロパティ構造を前提とする

## 2.4 要件定義との関係

本仕様群は、要件定義に基づいて構築される。

- 要件定義は「何を実現するか」を定義する
- 本仕様は「どのように実現するか」を定義する

仕様の各要素は、要件定義の以下に対応する。

- モデル構造 → メタモデル仕様
- 権利・許諾 → Rights / Validator
- モジュール構成 → Module / Slot
- 状態制御 → State / Core API
- 経済圏 → Revenue
- Runtime動作 → Runtime（非スコープ）

## 4. 仕様間の関係

仕様間の依存関係は以下の通り。

```text
メタモデル
   ↓
glTF拡張
   ↓
JSON Schema
   ↓
Validator
   ↓
Core API
   ↓
変換仕様
```

### 4.1 補足

- `glTF拡張` は `メタモデル` の外部表現
- `JSON Schema` は `glTF拡張` の検証形式
- `Validator` は `メタモデル` / `glTF拡張` / `JSON Schema` の意味検証
- `Core API` は `メタモデル` を操作する API
- `変換仕様` は `メタモデル` / `glTF拡張` を相互変換する

---

## 5. 設計原則

SansaVRM は以下の設計原則に基づく。

### 5.1 分離

以下の要素は明確に分離される。

- 構造（Model）
- 接続（Slot）
- 状態（State）
- 権利（Rights）
- 収益（Revenue）

---

### 5.2 非可逆対応

- 変換不能な情報は `source_raw` に保存する
- 変換時の問題は `diagnostics` に記録する

---

### 5.3 ID中心設計

- すべての要素は ID により識別される
- 参照整合性は Validator により保証される

---

### 5.4 Hub構造

- すべての外部フォーマットとの変換は SansaVRM Core を経由する

---

## 6. 実装前提

実装は以下を前提とする。

- JSON Schema による構造検証が可能であること
- Validator による整合性検証が行われること
- Core API を通じてモデル操作が行われること
- 外部フォーマットとの変換は Adapter を介すること

---

## 7. テスト仕様との関係

テスト仕様は以下の構成で別ディレクトリに定義する。

```text
ja-JP/03_テスト仕様/
```

テストは以下を対象とする。

- JSON Schema 検証
- Validator 検証
- Core API 動作
- 変換処理
- ラウンドトリップ整合性

---

## 8. 非スコープ

本仕様は以下を対象外とする。

- レンダリング処理
- 物理演算
- UI 表示
- ネットワーク同期
- 実行環境依存処理

---

本仕様群により、SansaVRM は以下を満たす。

- 検証仕様

- 変換仕様

これにより、実装可能な仕様体系として成立する。
