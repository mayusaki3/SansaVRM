# 変換仕様

## 1. 目的

本仕様は、SansaVRM Format と外部フォーマット（glTF / VRM / URDF）との相互変換に関する仕様を定義する。

本仕様の目的は以下とする。

- 外部フォーマットから SansaVRM Core への変換を定義する
- SansaVRM Core から外部フォーマットへの出力方法を定義する
- 非可逆変換の扱いを明確にする
- 拡張情報の保持方法を統一する
- Validator と連携した変換の正当性を担保する

---

## 2. 基本方針

- SansaVRM を中間表現（Hub）とする
- すべての変換は SansaVRM Core を経由する
- 外部フォーマットの情報は可能な限り保持する
- 変換不能な情報は `source_raw` に保存する
- 非可逆変換は `diagnostics` に記録する
- 変換後は必ず Validator を実行する
- glTF は物理表現、SansaVRM は論理表現とする

変換は原則として Model 単位で行う。
ただし、部分シリアライズが有効な場合は、
Module / Slot / State 単位での部分変換を許容する。

---

## 3. 対象フォーマット

本仕様では以下を対象とする。

- glTF 2.0
- VRM 0.x / 1.x
- URDF

---

## 4. 変換アーキテクチャ

```text
外部フォーマット
   ↓
Importer（Adapter）
   ↓
SansaVRM Core（内部モデル）
   ↓
Exporter（Adapter）
   ↓
外部フォーマット
```

---

## 5. 変換フェーズ

変換は以下のフェーズで構成する。

1. パース
2. 中間構造生成
3. メタモデルマッピング
4. 拡張情報抽出
5. source_raw格納
6. diagnostics生成
7. Validator実行

外部フォーマットから SansaVRM へ変換する際の ID は以下の規則で生成する。

- 既存の一意識別子が存在する場合はそれを優先する
- 存在しない場合は UUID またはパス形式 ID を生成する
- 同一入力からは同一IDが生成されることが望ましい（安定性）

---

## 6. glTF → SansaVRM

### 6.1 基本方針
<!-- hldocs:sec_id=sec_v8u6x1l0 -->

- glTF ノード構造を Module / Connection に変換する（node.children を Connection として直接マッピングする）
- Mesh / Material は glTF_binding に保持
- extensions に SansaVRM が存在する場合は優先的に利用

---

### 6.2 マッピング
<!-- hldocs:sec_id=sec_v8u6x1l1 -->

| glTF | SansaVRM |
|------|----------|
| node | Module |
| node.children | Connection（各 child が1つの Connection に対応） |
| mesh | glTF_binding.mesh_index |
| material | glTF_binding.material_indices |
| animation | glTF_binding.animation_indices |

---

### 6.3 接続生成
<!-- hldocs:sec_id=sec_v8u6x1l2 -->

- glTF の node.children は直接 Connection としてマッピングする
- from_id は親 node、to_id は child node とする

生成する Connection は以下を持つ：

- connection_id
- from_id
- to_id
- connection_type = attach
- enabled = true

---

### 6.4 未対応情報
<!-- hldocs:sec_id=sec_v8u6x1l3 -->

以下は source_raw に保存する。

- カスタム extensions
- 未解釈の metadata

---

## 7. VRM → SansaVRM

### 7.1 基本方針
<!-- hldocs:sec_id=sec_w7v5y0m1 -->

- VRM は glTF ベースのため glTF → SansaVRM を前段に使用
- VRM 固有情報は Extension Layer に保持

---

### 7.2 マッピング
<!-- hldocs:sec_id=sec_w7v5y0m2 -->

| VRM | SansaVRM |
|-----|----------|
| humanoid | Module (Body) |
| blendshape | State (Expression 系 State として分類) |
| firstPerson | Property |
| meta | Rights / Property |

---

### 7.3 権利情報
<!-- hldocs:sec_id=sec_w7v5y0m3 -->

VRM meta は以下に変換する。

- `normalized` にマッピング
- 元データは `source_raw` に保存

---

### 7.4 非可逆情報
<!-- hldocs:sec_id=sec_w7v5y0m4 -->

- license の一部表現
- custom metadata

→ diagnostics に記録

---

## 8. URDF → SansaVRM

### 8.1 基本方針
<!-- hldocs:sec_id=sec_x6w4z9n2 -->

- link を Module に変換
- joint を Connection に変換
- 物理情報は Property として保持

本仕様では接続関係を Connection により一元管理する。

Slot は以下の情報を保持する補助構造とする。

- 接続制約（target_slot_types, max_connections, exclusive 等）
- 接続可能なインターフェース定義
- 物理・制御などの役割分類（必要に応じて）

接続関係そのものは Connection のみが保持する。

---

### 8.2 マッピング
<!-- hldocs:sec_id=sec_x6w4z9n3 -->

- URDF の joint は parent → child の方向で Connection に変換する

| URDF | SansaVRM |
|------|----------|
| link | Module |
| joint | Connection (joint) |
| inertial | Property |
| visual | glTF_binding |
| collision | Property |

---

### 8.3 制約
<!-- hldocs:sec_id=sec_x6w4z9n4 -->

- joint type を constraints に変換
- limit を Property に変換

---

## 9. SansaVRM → glTF

### 9.1 基本方針
<!-- hldocs:sec_id=sec_y5x3a8p3 -->

- Module を node に変換
- connections は以下の優先順位で反映する。
 1. 木構造として表現可能な場合は node の親子関係として出力する
 2. 木構造で表現できない場合は glTF extensions に保持する
- SansaVRM は glTF extensions に格納

---

### 9.2 出力構造
<!-- hldocs:sec_id=sec_y5x3a8p4 -->

```json
{
  "asset": { "version": "2.0" },
  "nodes": [...],
  "extensions": {
    "SansaVRM_model": {...},
    ...
  }
}
```

---

## 10. SansaVRM → VRM 0.X / 1.0

### 10.1 方針
<!-- hldocs:sec_id=sec_z4y2b7q4 -->

- glTF 出力をベースとする
- VRM meta を生成
- Rights から license を再構築

---

## 11. SansaVRM → URDF

### 11.1 方針
<!-- hldocs:sec_id=sec_a3z1c6r5 -->

- Module を link に変換
- connection_type = joint の Connection を joint に変換
- Property および補助構造から inertial / collision を再構築
- Control / Actuator / Sensor 情報は Property または Extension として出力

URDF への Property 出力分類は以下とする。

- Physics + Simulation / Execution → inertial
- Geometry + Rendering → visual
- Geometry + Simulation → collision
- Material / Texture + Rendering → visual
- その他 → URDF XML へ直接出力しない

---

## 12. 非可逆変換

### 12.1 原則

完全な再現ができない場合は以下で対応する。

- `source_raw` に元データ保存
- `diagnostics` に記録

---

### 12.2 例

- VRM meta → SansaVRM → VRM で完全一致しない
- URDF → SansaVRM → URDF で joint 情報が欠落

---

## 13. Validator連携

- Import 後に必ず validate
- Export 前に validate
- エラーがある場合は出力禁止（strictモード）

---

## 14. エラー処理

- パースエラー → error (error_code は Validator と揃える)
- マッピング不能 → warning
- 情報欠落 → info

---

## 15. オプション

- `strict`
- `preserve_raw`
- `lossy_allowed`
- `target_format`

---

## 16. Adapter責務

Importer / Exporter は以下を担う。

- フォーマット解析
- メタモデル変換
- source_raw 保存
- diagnostics 生成

---

## 17. 非スコープ

- Renderer
- Runtime制御
- UI
- ネットワーク同期
