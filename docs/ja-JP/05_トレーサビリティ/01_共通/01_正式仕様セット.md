# 正式仕様セット

## 1. 目的

本ドキュメントは、SansaVRM の traceability 対象となる正式仕様セットを固定するための基準を定義する。

Phase 0 の完了までは、本ドキュメントを暫定正本として扱う。

---

## 2. 正式仕様セットの原則

正式仕様セットは、以下の条件を満たす文書で構成する。

```text
1. docs/ja-JP/目次.md から到達できる
2. canonical_document: true を持つ
3. SansaVRM Core、外部フォーマット連携、検証、テスト、実装計画のいずれかに直接関係する
4. 草案、実験、廃止予定として明示されていない
```

---

## 3. 文書分類

文書は以下に分類する。

| 分類 | 意味 | traceability 対象 |
|---|---|---|
| 正式仕様 | SansaVRM の仕様判断に使用する文書 | 対象 |
| Core Semantic | SansaVRM Core の意味論を定義する文書 | 対象 |
| テスト仕様 | 正式仕様から導出される検証文書 | 対象 |
| 実装計画 | 実装順序と作業方針を定義する文書 | 原則対象外。ただし判断根拠として参照可 |
| validator専用 | validator platform の実装・検証に限定される文書 | Core 固定後に再評価 |
| adapter専用 | 特定 adapter の実装詳細に限定される文書 | 対象。ただし Core Semantic より下位 |
| 草案 | 正式採用前の検討文書 | 対象外 |
| 実験 | PoC または実験用文書 | 対象外 |
| 廃止予定 | 今後削除または置換する文書 | 対象外 |

---

## 5. 現時点の判断保留対象

以下は正式仕様候補であるが、Phase 0 の棚卸し完了までは対象範囲を再評価する。

```text
02_仕様/01_共通/05_Validator実装仕様.md
02_仕様/01_共通/09_MuJoCo連携仕様.md
03_テスト仕様/01_共通/03_Validatorテスト仕様.md
```

理由は、validator platform と MuJoCo schema-driven validation が現在先行しており、SansaVRM Core の意味論固定前に traceability 対象として広げすぎると、仕様集合が不安定化するためである。

---

## 6. Core Semantic として新設すべき文書

Phase 0 では、以下を新設または分離する。

```text
02_仕様/01_共通/10_Core Semantic Definition.md
02_仕様/01_共通/11_Semantic Preservation Matrix.md
```

これらは、VRM I/O と RoundTrip 保証の前提となる。

---

## 7. traceability 対象固定ルール

traceability 対象は、以下の順で固定する。

```text
1. Core Semantic
2. VRM I/O
3. RoundTrip
4. Core Property
5. URDF
6. MuJoCo Runtime / Adapter
7. Validator Platform
```

Validator Platform は、Core Semantic と VRM I/O の意味論が安定するまで拡張を凍結する。

---

## 8. 完了条件

Phase 0 の完了条件は以下である。

```text
1. docs/ja-JP/目次.md から正式仕様へ到達できる
2. 正式仕様候補と判断保留対象が分離されている
3. Core Semantic 文書が追加されている
4. Semantic Preservation Matrix が追加されている
5. traceability 対象の初期集合が明文化されている
```
