# 開発環境構築

## 1. 目的

本ドキュメントは、SansaVRM の実装に必要な開発環境および初期セットアップ手順を定義する。

本ドキュメントは、すべての実装ロードマップの前提となる環境を定義するものであり、常に最新状態を維持する正本とする。

---

## 2. 前提

- OS：Windows / Linux / macOS（いずれも可）
- Git が使用可能であること
- インターネット接続が可能であること

---

## 3. 使用技術

- Rust（Core / Validator / Adapter 実装）
- JSON Schema
- glTF / VRM / URDF
- GitHub（リポジトリ / CI）

---

## 4. Rust 環境構築

### 4.1 インストール

以下コマンドで Rust をインストールする。

```bash
curl https://sh.rustup.rs -sSf | sh
```

Windows の場合は公式インストーラを使用する。

---

### 4.2 確認

```bash
rustc --version
cargo --version
```

---

### 4.3 ツールチェーン

```bash
rustup update
rustup default stable
```

---

## 5. プロジェクト作成

### 5.1 ルート確認（または作成）

本プロジェクトは Git リポジトリを前提とする。

- 新規作成の場合：
```bash
  mkdir SansaVRM
  cd SansaVRM
  git init
```

- 既存リポジトリを使用する場合：
  リポジトリルートで後続の手順を実行する

---

### 5.2 Cargo workspace

リポジトリルートに `Cargo.toml` を作成し、workspace を定義する。

#### ファイル作成

- Linux / macOS:

```bash
touch Cargo.toml
```

- Windows（PowerShell）:

```powershell
New-Item Cargo.toml
```

- Windows（コマンドプロンプト）:

```cmd
type nul > Cargo.toml
```

#### 設定内容

```toml
[workspace]
resolver = "2"
members = [
  "crates/sansavrm-core",
  "crates/sansavrm-validator",
  "crates/sansavrm-gltf",
  "crates/sansavrm-vrm",
  "crates/sansavrm-urdf"
]
```

※ 上記の crates は後続手順で作成する。

---

### 5.3 crate 作成

```bash
mkdir crates

cargo new crates/sansavrm-core --lib
cargo new crates/sansavrm-validator --lib
cargo new crates/sansavrm-gltf --lib
cargo new crates/sansavrm-vrm --lib
cargo new crates/sansavrm-urdf --lib
```

---

### 5.4 共通ディレクトリ作成

以下のディレクトリを作成する。

- Linux / macOS:

```bash
mkdir schemas tests samples
mkdir -p tests/schema tests/validator tests/core tests/convert tests/roundtrip
mkdir -p samples/gltf samples/vrm samples/urdf
```

- Windows（PowerShell）:

```powershell
New-Item -ItemType Directory schemas, tests, samples
New-Item -ItemType Directory tests/schema, tests/validator, tests/core, tests/convert, tests/roundtrip
New-Item -ItemType Directory samples/gltf, samples/vrm, samples/urdf
```

- Windows（コマンドプロンプト）:

```cmd
mkdir schemas tests samples
mkdir tests\schema tests\validator tests\core tests\convert tests\roundtrip
mkdir samples\gltf samples\vrm samples\urdf
```

## 10. CI（GitHub Actions）

### 例

```yaml
name: Rust CI

on:
  push:
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Rust
        uses: actions-rs/toolchain@v1
        with:
          toolchain: stable
      - run: cargo test
```
