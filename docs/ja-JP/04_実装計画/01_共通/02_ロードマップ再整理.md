# ロードマップ再整理

## 1. 目的

本ドキュメントは、SansaVRM 開発において先行した MuJoCo validator / schema-driven validation platform の成果を一旦凍結し、SansaVRM Core Semantic および外部フォーマット統合に必要な実データモデルを優先するための再整理方針を定義する。

---

## 2. 現在の状態

現在の開発では、本来の優先順よりも MuJoCo validator / schema-driven validation platform が先行している。

当初の優先順は以下であった。

```text
VRM I/O 完成
↓
RoundTrip保証
↓
Core Property安定化
↓
URDF
↓
MuJoCo
↓
Validator Platform
```

一方、実際には以下が先行している。

```text
MuJoCo validator / schema-driven validation platform
```

その結果、validator platform は強いが、VRM / FBX / MMD / URDF / MuJoCo を統合するための実データモデルが未完成である。

---

## 3. 完了済み成果

MuJoCo Schema-Driven Validation Platform は、PoC として以下を実装済みである。

```text
- schema-driven validator
- diagnostics
- conversion report
- adapter artifact separation
- fallback handling
- preserve_only / unsupported / source_raw
- traceability matrix
- traceability coverage analysis
- JSON Schema validation
- schema evolution compatibility checker
- GitHub Actions CI
- formal schemas
- traceability manifest
```

また、仕様面では以下を追加済みである。

```text
- Core Semantic Definition
- Semantic Preservation Matrix
- RoundTrip Semantic Criteria
- Adapter Extension Property Specification
- Format Compatibility Preservation Specification
- 正式仕様セット
```

これにより、SansaVRM の semantic foundation は整理された。

一方で、Geometry / Rig / Skinning / Morph / Animation / Physics の実体定義はまだ不足している。

---

## 4. 現在の主要問題

### 4.1 実データモデル未完成

SansaVRM が保持すべき semantic は整理されたが、以下の実体モデルが未完成である。

```text
- Geometry
- Rig
- Skinning
- Morph
- Animation
- Physics abstraction
- Conversion Profile
```

これらが未定義のまま Adapter 実装へ戻ると、VRM / FBX / MMD / URDF / MuJoCo 間の変換基盤が不安定になる。

---

### 4.2 VRM I/O 未完成

VRM 0.x / 1.0 import / export が未完成である。

特に以下を優先する必要がある。

```text
- humanoid
- expression
- spring bone
- lookAt
- meta
- extension passthrough
```

ただし、VRM I/O の実装へ戻る前に、Geometry / Rig / Morph / Animation / Physics の共通実体を整理する必要がある。

---

### 4.3 FBX / MMD 互換性未整理

FBX / MMD は、VRM や MuJoCo と異なり、元形式へ戻すための Format Preservation が重要である。

そのため、以下が必要である。

```text
- Format Preservation Layer
- Loss Report
- Conversion Profile
- raw_binary_ref
- Geometry / Rig / Skinning / Morph / Animation 実体
```

---

### 4.4 Validator Platform 先行問題

SansaVRM の本来の価値は、validator platform 単体ではなく、以下の統合である。

```text
VRM / FBX / MMD / URDF / MuJoCo 統合
```

そのため、validator platform は SansaVRM Core Semantic と実データモデルが安定した後に再開する。

---

## 5. 新ロードマップ

## Phase 0：ドキュメント再編・正式仕様セット固定

### 目的

正式仕様集合を固定し、traceability 対象を明確化する。

### 作業

```text
1. docs/ja-JP/目次.md を正規化する
2. 未リンク文書を棚卸しする
3. 正式仕様、草案、実験、adapter専用、validator専用、廃止予定を分類する
4. traceability 対象仕様を固定する
5. 旧階層と新階層が混在している場合、正式な配置先を決定する
```

### 現在の状態

```text
一部完了
```

追加済み：

```text
- 正式仕様セット
- Core Semantic Definition
- Semantic Preservation Matrix
- RoundTrip Semantic Criteria
- Adapter Extension Property Specification
- Format Compatibility Preservation Specification
```

---

## Phase 1：Geometry / Rig / Skinning Extension

### 目的

VRM / FBX / MMD の共通基盤となる Geometry / Rig / Skinning 実体を定義する。

### 作業

```text
- mesh
- submesh
- vertex attribute
- material slot
- bone hierarchy
- bind pose
- inverse bind matrix
- skinning method
- MMD BDEF / SDEF / QDEF preservation
```

### 完了条件

```text
- Geometry/Rig/Skinning の仕様が定義されている
- Format Preservation Layer と接続できる
- VRM / FBX / MMD adapter が参照できる
```

---

## Phase 2：Morph Extension

### 目的

VRM expression、FBX BlendShape、MMD morph を共通的に扱うための Morph 実体を定義する。

### 作業

```text
- morph target
- morph type
- weight range
- material morph
- bone morph
- UV morph
- group morph
- expression mapping
```

### 完了条件

```text
- Expression Semantic と Morph 実体の関係が定義されている
- MMD / FBX / VRM 間の morph 変換方針が定義されている
```

---

## Phase 3：Animation Extension

### 目的

FBX / MMD / VRM の animation 実体を保持・変換するための Animation Extension を定義する。

### 作業

```text
- animation clip
- channel
- curve
- interpolation
- timeline
- frame rate
- time unit
- FBX stack / layer / tangent preservation
```

### 完了条件

```text
- Animation 実体が定義されている
- Format Preservation Layer と接続できる
- loss_report に animation loss を記録できる
```

---

## Phase 4：Physics Extension

### 目的

MMD physics、VRM spring bone、URDF physical model、MuJoCo physics を扱うための Physics abstraction を定義する。

### 作業

```text
- rigid body
- collider
- joint
- constraint
- collision group
- spring
- backend-specific physics extension
```

### 完了条件

```text
- Physics Runtime 非依存の抽象化が定義されている
- Backend 固有情報を Compatibility Extension へ退避できる
- MuJoCo / URDF / MMD / VRM spring bone と接続できる
```

---

## Phase 5：Conversion Profile Specification

### 目的

外部フォーマット間の mapping ルールを定義する。

### 作業

```text
- bone mapping
- coordinate mapping
- unit mapping
- expression mapping
- morph mapping
- material mapping
- physics mapping
- animation mapping
```

### 完了条件

```text
- MMD → VRM
- FBX → VRM
- VRM0 ↔ VRM1
- URDF → MuJoCo
```

などの変換で使用する profile を定義できる。

---

## Phase 6：VRM I/O 完成

### 目的

以下の変換で意味的欠落なしを成立させる。

```text
VRM 0.x / 1.0
↓
SansaVRM
↓
VRM 0.x / 1.0
```

### 重点対象

```text
- humanoid
- expression
- spring bone
- lookAt
- meta
- extension passthrough
```

### 完了条件

```text
- VRM0 import/export
- VRM1 import/export
- semantic roundtrip tests
- extension passthrough tests
- loss_report integration
```

---

## Phase 7：FBX / MMD Compatibility Adapter

### 目的

FBX / MMD を SansaVRM の Format Compatibility Preservation 方針に基づき読み書きする。

### 作業

```text
- FBX import/export profile
- MMD import/export profile
- raw preservation
- raw_binary_ref
- loss_report
- morph / animation / physics mapping
```

### 完了条件

```text
- MMD → SansaVRM → MMD
- FBX → SansaVRM → FBX
- MMD → SansaVRM → VRM
- FBX → SansaVRM → VRM
```

の最低限の変換判定が可能である。

---

## Phase 8：URDF 強化

### 目的

Geometry / Rig / Physics が安定した後、URDF 連携を強化する。

URDF は構造・物理寄りのフォーマットであるため、Physics Extension および Conversion Profile を前提に扱う。

---

## Phase 9：MuJoCo Runtime / Adapter

### 目的

現在の MuJoCo validator platform の成果を活用し、実際の MuJoCo runtime / adapter 連携へ進める。

### 不足対象

```text
- real MJCF emitter
- actuator emission
- sensor emission
- runtime mapping
- sysid integration
- HIL integration
- Adapter Extension Property integration
```

---

## Phase 10：Validator Platform 再開

### 目的

SansaVRM Core Semantic、実データモデル、Format Compatibility、Conversion Profile が固定された後、validator platform の拡張を再開する。

### 対象

```text
- validator code generation
- schema bootstrap
- migration checker
- compatibility checker
- semantic diff
```

### 注意点

仕様固定前に code generation を進めると、仕様変更時の作り直しが大きくなるため、Phase 10 まで後回しにする。

---

## Phase Final：HLDocS feedback issue 作成

### 目的

今回の SansaVRM 仕様体系再配置、Layer refactor、Traceability migration、dry-run relocation から得られた知見を HLDocS 側へ還元する。

### 作業

```text
- HLDocS リポジトリに feedback issue を作成する
- doc_id / sec_id migration policy の知見を整理する
- parallel structure migration の有効性を整理する
- migration manifest / dry-run manifest の必要性を整理する
- Layer semantic locality と軽微な重複許容の扱いを整理する
- hierarchy refactor 時の breadcrumb / link 更新方針を整理する
- semantic equivalence tracking の必要性を整理する
- migration validator / CI integration の必要性を整理する
```

### 完了条件

```text
- HLDocS 側に issue が作成されている
- SansaVRM で実施した migration workflow が feedback として記録されている
- HLDocS 側で仕様改善候補として追跡可能になっている
```

---

## 6. 凍結対象

以下は現時点で PoC として十分なため、一旦凍結する。

```text
tools/mujoco_schema_validator/
schemas/mujoco/schema_driven_validation/
traces/mujoco/schema_driven_validation/
```

凍結中は、以下を原則として行わない。

```text
- validator code generation の追加
- schema bootstrap の追加
- migration checker の高度化
- MuJoCo validator platform 単体の機能拡張
```

ただし、既存 CI を壊さないための保守修正は許可する。

## 8. 判断基準

今後の作業優先度は以下で判断する。

| 優先 | 対象 | 判断 |
|---|---|---|
| 1 | Geometry / Rig / Skinning | 外部フォーマット統合の実体基盤 |
| 2 | Morph | VRM expression / MMD morph / FBX BlendShape の統合基盤 |
| 3 | Animation | FBX / MMD / VRM animation 保持に必要 |
| 4 | Physics | URDF / MuJoCo / MMD / VRM spring bone 連携に必要 |
| 5 | Conversion Profile | 形式間変換の再現性に必要 |
| 6 | VRM I/O | SansaVRM の主要入口として必要 |
| 7 | FBX / MMD Adapter | Format Compatibility の実証 |
| 8 | URDF | 構造・物理フォーマット連携 |
| 9 | MuJoCo Runtime / Adapter | 物理・制御・Runtime 連携 |
| 10 | Validator Platform | 仕様固定後に再開 |
| Final | HLDocS feedback issue | 仕様移行ワークフローの知見還元 |

---

## 9. 結論

現在の MuJoCo validator / schema-driven validation platform は、PoC として十分に強い状態に到達している。

また、SansaVRM の semantic foundation は以下により整理された。

```text
- Core Semantic Definition
- Semantic Preservation Matrix
- RoundTrip Semantic Criteria
- Adapter Extension Property Specification
- Format Compatibility Preservation Specification
```

今後は validator platform の拡張を一旦凍結し、Geometry / Rig / Skinning / Morph / Animation / Physics / Conversion Profile の実データモデルを優先する。

これにより、SansaVRM を単なる VRM 派生ではなく、VRM / FBX / MMD / URDF / MuJoCo を接続する semantic integration platform として成立させる。
