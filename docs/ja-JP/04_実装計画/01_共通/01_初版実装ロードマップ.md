# 初版実装ロードマップ

## 1. 目的

本ドキュメントは、SansaVRM の仕様およびテスト仕様に基づき、初版実装作業の順序と成果物を定義する。（実装完了後は履歴として保持する）

---

## 2. 実装方針

- 仕様を先に固定し、実装は仕様に従う
- テスト仕様に基づいてテストを先に作成する
- Rust Core を中心に実装する
- glTF / VRM / URDF は Adapter として分離する
- 実装後は必ず Validator とテストで検証する

---

## 3. 実装フェーズ

## Phase 1：プロジェクト基盤

### 目的

開発・テスト・CI の基盤を整備する。

### 作業

- Rust プロジェクト作成
- crate 構成定義
- CI 設定
- テストディレクトリ作成
- サンプルデータ配置

### 成果物

- Cargo プロジェクト
- CI 設定
- テスト実行基盤

---

## Phase 2：メタモデル実装

### 目的

`02_メタモデル仕様.md` に基づく内部データ構造を実装する。

### 作業

- Model 実装
- Module 実装
- Slot 実装
- State 実装
- Rights / Revenue 実装
- diagnostics 実装

### 成果物

- Core data model
- 基本データ構造の単体テスト

---

## Phase 3：Validator 実装

### 目的

`05_Validator実装仕様.md` に基づく意味検証を実装する。

### 作業

- ID 一意性検証
- 参照整合性検証
- connections / current_connections 検証
- StateAction 検証
- owner / ref 検証
- diagnostics 生成

### 成果物

- Validator
- Validator テスト

---

## Phase 4：JSON Schema 実装

### 目的

`04_JSONスキーマ仕様.md` に基づき、Schema 検証を実装する。

### 作業

- schema ファイル分割
- root schema 作成
- defs schema 作成
- 個別 schema 作成
- Schema テスト作成

### 成果物

- JSON Schema 一式
- JSON Schema テスト

---

## Phase 5：Core API 実装

### 目的

`06_CoreAPI仕様.md` に基づくモデル操作 API を実装する。

### 作業

- Model 管理 API
- Module 操作 API
- Slot 操作 API
- Connection 操作 API
- State 操作 API
- Evaluate API
- Transaction API

### 成果物

- Core API
- Core API テスト

---

## Phase 6：glTF Adapter 実装

### 目的

glTF と SansaVRM の相互変換を実装する。

### 作業

- glTF Importer
- glTF Exporter
- SansaVRM glTF extensions 出力
- glTF binding 処理
- 変換テスト

### 成果物

- glTF Adapter
- glTF 変換テスト

---

## Phase 7：VRM Adapter 実装

### 目的

VRM 0.x / 1.x と SansaVRM の相互変換を実装する。

### 作業

- VRM Importer
- VRM Exporter
- VRM meta → Rights 変換
- blendshape → State 変換
- source_raw / diagnostics 処理

### 成果物

- VRM Adapter
- VRM 変換テスト

---

## Phase 8：URDF Adapter 実装

### 目的

URDF と SansaVRM の相互変換を実装する。

### 作業

- URDF Importer
- URDF Exporter
- link → Module 変換
- joint → Slot / Connection 変換
- inertial / visual / collision 変換

### 成果物

- URDF Adapter
- URDF 変換テスト

---

## Phase 9：ラウンドトリップ検証

### 目的

外部フォーマットとの往復変換品質を検証する。

### 作業

- glTF ラウンドトリップテスト
- VRM ラウンドトリップテスト
- URDF ラウンドトリップテスト
- 異フォーマット経由テスト
- diagnostics 差分確認

### 成果物

- ラウンドトリップテスト
- 変換品質レポート

---

## Phase 10：最小リリース

### 目的

最小機能を備えた初期版を作成する。

### 最小対象

- SansaVRM Core
- JSON Schema
- Validator
- glTF Import / Export
- 基本テスト一式

### 成果物

- v0.1.0
- README 更新
- 利用例
- サンプルモデル

---

## 7. 結論

本ロードマップに従い、SansaVRM は以下の順で実装する。

- 仕様に基づくデータ構造
- 自動検証
- Core API
- 外部フォーマット変換
- ラウンドトリップ検証
- 最小リリース

これにより、仕様・テスト・実装が一貫した開発体制を構築する。
