# モジュール構成・装着

## 1. 目的

本仕様は、SansaVRM におけるモジュール構成および装着方式を定義する。

SansaVRM は、単一の完成モデルを扱うだけでなく、
複数のモデル要素を組み合わせて1つの利用対象を構成する
「構成可能なVRモデル基盤」として動作する。

本仕様は、衣装・小物・装備・モジュール部品・ワールドオブジェクト等を、
共通的な装着・接続モデルで扱うための基準を定める。

---

## 2. 基本方針

- モデルは単体でも複合でも扱えること
- 装着はノード接続ではなく、意味を持つ構造接続として扱う
- 人型専用に限定しない
- URDF 相当の構造接続思想を取り込めること
- glTF / VRM / URDF 等の入力形式差異を内部表現で吸収すること
- 装着時には構造適合性および権利・許諾条件の両方を確認できること
- 装着構造は Runtime で再構成可能であること

---

## 3. モジュール構成の考え方

SansaVRM におけるモデルは、以下のいずれかで構成される。

- 単体モデル
- 複合モデル

単体モデルは、1つのモデル単位で完結する構成をいう。  
複合モデルは、複数のモジュールを接続・装着して成立する構成をいう。

複合モデルにおいては、各構成要素は独立したモデル単位として保持される。

---

## 4. 構成単位

SansaVRM は、以下の構成単位を扱う。

- Body
- Clothing
- Accessory
- Equipment
- Prop
- Module
- World Object
- Composite Root

### 4.1 Body

本体となる中心モデル。  
アバター、ロボット本体、クリーチャー本体などを含む。

### 4.2 Clothing

衣服、外装、被覆物。  
装着対象の外観変化を主目的とするモジュール。

### 4.3 Accessory

アクセサリ、小物、装飾品。  
比較的小規模で、特定部位へ装着されるモジュール。

### 4.4 Equipment

装備品。  
武器、工具、防具、外部機構等、役割を持つモジュール。

### 4.5 Prop

持ち物、手持ちアイテム、設置可能物。  
Body に装着される場合も、World に配置される場合もある。

### 4.6 Module

構造部品、交換部品、追加機能部品。  
ロボット部品、追加脚、補助アーム、換装ユニット等を含む。

### 4.7 World Object

ワールド側に存在する独立オブジェクト。  
家具、建築物、装置、ギミック等を含む。

### 4.8 Composite Root

複数モジュールから構成される複合体のルート。  
複合モデル全体を管理する仮想的または実体的な親要素。

---

## 5. 装着・接続モデル

SansaVRM における装着は、単純な親子付けではなく、
接続規則を持つ構造関係として扱う。

装着・接続は Equipment Slot を用いて定義する。

### 5.1 接続の基本概念

1. 接続元モデルが存在する
2. 接続先モデルが存在する
3. 接続元は装着点を持つ
4. 接続先は受け入れ可能な接続条件を持つ
5. 両者の条件が一致した場合に接続が成立する

### 5.2 接続対象

- Body ← Clothing
- Body ← Accessory
- Body ← Equipment
- Body ← Prop
- Body ← Module
- Module ← Module
- World Object ← Prop
- World Object ← Module
- Composite Root ← 任意の構成要素

---

## 6. Equipment Slot

Equipment Slot は、装着・接続構造を定義するための Slot である。

### 6.1 Equipment Slot の責務

- 装着点の定義
- 装着対象の定義
- 位置合わせの定義
- 接続制約の定義
- Runtime 再構成時の接続情報提供

### 6.2 Equipment Slot の代表項目

- attach_point
- target_node
- target_slot
- offset_transform
- required_rig
- allowed_targets
- excluded_targets
- attach_mode
- detach_policy

### 6.3 attach_point

接続元が提供する装着点識別子。  
例:

- head_top
- chest_front
- back_mount
- left_hand
- waist_left
- tail_base
- module_port_a

### 6.4 target_node

接続先モデル内で接続対象となるノード。  
既存ノード、仮想ノード、構造ノードを指定可能とする。

### 6.5 target_slot

接続先モデルが持つ受け入れスロット。  
attach_point と target_slot の対応により接続成立を判定する。

### 6.6 offset_transform

接続成立時に適用する相対変換。  
位置、回転、スケールの補正を含む。

### 6.7 required_rig

接続に必要な構造条件。  
特定ボーン、ジョイント、リンク構造を要求できる。

### 6.8 allowed_targets

装着可能対象の一覧。  
例:

- humanoid
- robot
- world_object
- body
- module_port

### 6.9 excluded_targets

装着禁止対象の一覧。  
例:

- creature
- quadruped
- fragile_mount

### 6.10 attach_mode

装着方式。  
少なくとも以下を想定する。

- fixed
- follow
- articulated
- replace
- merge

#### fixed
固定接続。  
接続後、接続先と一体として扱う。

#### follow
追従接続。  
接続先の移動や変形に追従する。

#### articulated
関節付き接続。  
接続後も一定の可動性を持つ。

#### replace
既存部位または既存モジュールを置き換える接続。

#### merge
接続後、1つの複合構造として統合される接続。

### 6.11 detach_policy

取り外し時の動作方針。  
例:

- free_detach
- restricted_detach
- runtime_locked
- destroy_on_detach

---

## 7. URDF 相当の構造接続

SansaVRM は URDF を直接そのまま内部構造にするのではなく、
URDF 的な「構造体同士の関係定義」を取り込めるようにする。

### 7.1 採用する考え方

- 部位を構造単位として扱う
- 部位間を接続関係で表現する
- 接続には意味を持たせる
- 固定関節と可動関節を区別できる
- 機構部品の追加・交換を許容する

### 7.2 取り込む要素

- link 相当の構造単位
- joint 相当の接続関係
- parent / child 構造
- 可動制約
- 接続位置

### 7.3 取り込まない前提

- SansaVRM は URDF そのものに固定しない
- URDF の全項目を必須にはしない
- フォーマット固有構造は Format Extension Layer で保持する

---

## 8. 装着成立条件

装着は、以下の条件をすべて満たした場合に成立する。

1. 接続先が存在する
2. attach_point と target_slot の対応が有効である
3. Compatibility Slot に違反しない
4. Rights Slot に違反しない
5. Runtime 上の制約に違反しない

いずれかを満たさない場合、装着は成立しない。

---

## 9. 装着時の検証項目

装着時には少なくとも以下を検証する。

- 接続対象の存在確認
- target_node の存在確認
- target_slot の受理可否
- required_rig の充足確認
- allowed_targets / excluded_targets の判定
- attach_mode の適用可否
- Compatibility 条件の充足確認
- Rights 条件の充足確認

---

## 10. 構成モデルの再構成

SansaVRM は Runtime において構成変更を許容する。

### 10.1 再構成の例

- 衣装差し替え
- 小物追加
- 武器装備
- 部品交換
- モジュール追加
- 一部取り外し
- ワールドオブジェクトへの再接続

### 10.2 再構成時の要求

- 接続整合性を再評価すること
- 権利・許諾条件を再評価すること
- 必要に応じて既存構成との競合を解消すること
- 構成変更前後の diagnostics を取得可能であること

---

## 11. 装着競合

複数モジュールが同一の装着点または競合する領域を要求する場合、
競合解決を行う必要がある。

### 11.1 競合要因

- 同一 attach_point の重複使用
- replace 対象の重複
- excluded_targets との衝突
- Rights 条件による利用不可
- Compatibility 条件による同時装着不可

### 11.2 競合結果

- 片方のみ許可
- 優先順位で決定
- 条件付き許可
- 両方拒否

---

## 12. 複合モデルと権利・許諾制御の連携

複合モデルでは、各モジュールが独立した Rights Slot を持つ。  
そのため、装着成立後も全体として利用可能かを再判定する必要がある。

装着構成は以下に影響する。

- 利用可否
- 表現制限
- 商用利用可否
- 再配布可否
- 収益分配対象
- 利用ログ

詳細は「03_権利・許諾制御」を参照する。

---

## 13. ワールドオブジェクトへの適用

SansaVRM の装着・接続モデルは、アバターだけでなく
ワールド側オブジェクトにも適用可能とする。

### 13.1 適用例

- 家具への装飾品設置
- 壁面へのパーツ追加
- 機械装置への部品換装
- ワールドギミックへの外部モジュール接続
- 展示物への小物配置

### 13.2 方針

- Body 専用の概念にしない
- World Object に対しても target_slot を定義可能とする
- アバターとワールドオブジェクトを同じ接続基盤で扱う

---

## 14. diagnostics

装着・接続時に問題がある場合、diagnostics に記録する。

### 14.1 記録対象例

- target_node 不在
- target_slot 不一致
- required_rig 不足
- excluded_targets 違反
- Compatibility 違反
- Rights 違反
- 非可逆構成変換
- attach_mode 非対応

### 14.2 用途

- ロード失敗理由の提示
- エディタ上での検証補助
- Runtime 警告表示
- 自動修正候補の提示
