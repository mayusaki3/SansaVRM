# Runtime動作

## 1. 目的

本仕様は、SansaVRM における Runtime 上での動作を定義する。

モデルは静的なデータではなく、
Runtime において動的に構成・評価・更新されることを前提とする。

---

## 2. 基本方針

- モデルは動的に構成可能である
- 判定（適合・権利）は Runtime で実行される
- 構成変更は即時反映可能である
- 部分更新が可能である
- ワールドオブジェクトと同一基盤で扱う
- Runtime は実行環境であり、処理ロジックは外部実装（Rust Core）に委譲する

---

## 3. Runtime構成要素

Runtime は以下の要素で構成される。

各構成要素は論理的な責務分離であり、
実装上は統合または分割される場合がある。

- モデル実体管理
- モジュール実体管理
- スロット管理機構
- 適合判定機構
- 権利判定機構
- 収益処理機構
- イベント管理機構
- 外部処理連携機構（Rust Core）

※各構成要素の具体的な実装方式およびクラス構造は、本仕様では定義しない。

---

## 4. モデル実体管理

モデル実体管理は、Runtime 上でのモデルの状態および構成を管理する。

### 4.1 保持情報

- 構成モジュール
- 接続関係
- 状態情報
- 権利状態
- 収益状態

---

## 5. モジュール実体管理

モジュール実体管理は、各モジュールの状態および接続情報を管理する。

### 5.1 保持情報

- 接続点情報
- 接続先情報
- 状態（有効 / 無効）
- ローカル設定

---

## 6. スロット管理機構

スロット管理機構は、スロットの登録、占有、解放および競合管理を行う。

### 6.1 機能

- スロット登録
- スロット占有管理
- スロット解放
- スロット競合検出

---

## 7. 適合判定機構

適合判定機構は、構造・タグ・スロット・制約に関する判定を行う。

### 7.1 機能

- 構造判定
- タグ判定
- スロット判定
- 制約判定

---

## 8. 権利判定機構

権利判定機構は、利用可否、条件評価および利用制御に関する判定を行う。

### 8.1 機能

- 利用可否判定
- 条件評価
- 利用制御

---

## 9. 収益処理機構

収益処理機構は、利用イベントに基づく分配計算およびログ記録を行う。

### 9.1 機能

- 利用イベント取得
- 分配計算
- ログ記録

---

## 10. イベント管理機構

イベント管理機構は、Runtime における処理をイベント駆動で制御する。

### 10.1 主なイベント

- onModelLoad
- onModuleAttach
- onModuleDetach
- onStructureChange
- onUsage
- onRightsCheck
- onRevenueProcess
- onStateChange
- onExpressionChange

---

## 11. 処理フロー

Runtime は実行環境として動作し、
判定・計算・構造処理などの中核ロジックは外部の処理系（Rust Core）に委譲する。

Runtime 側では、状態管理、イベント処理、実行環境との接続を担当する。

各処理ステップは逐次処理に限定されず、
並列または非同期で実行される場合がある。

### 11.1 モデルロード

1. モデル読み込み
2. モジュール情報取得
3. Rust Core による構造展開
4. スロット登録
5. Rust Core による適合判定
6. Rust Core による権利判定
7. 初期状態確定

---

### 11.2 モジュール装着

1. 装着要求
2. スロット確認
3. 適合判定
4. 権利判定
5. 装着実行
6. 状態更新

---

### 11.3 モジュール取り外し

1. 取り外し要求
2. 依存関係確認
3. 取り外し実行
4. 状態更新

---

### 11.4 利用イベント

1. 利用検知
2. 権利判定
3. 利用許可
4. 収益計算
5. ログ記録

### 11.5 状態切替

1. 状態切替要求
2. 状態識別子に対応する構成取得
3. Rust Core による構成妥当性確認
4. Rust Core による適合判定
5. Rust Core による権利判定
6. 状態反映
7. 必要に応じて利用状態更新

### 11.6 表現切替

1. 表現切替要求
2. 対象表現情報取得
3. Rust Core による表現妥当性確認
4. Morph / 材質 / テクスチャ状態更新
5. 状態反映

---

## 12. 状態管理

モデルは以下の状態を持つ。

- ACTIVE
- INACTIVE
- RESTRICTED
- INVALID

---

## 13. 再評価

構成変更時に再評価を行う。

### 13.1 対象

- モジュール追加
- モジュール削除
- 設定変更

---

### 13.2 結果

- 継続可能
- 制限付き
- 無効化

---

## 14. ワールドオブジェクト対応

Runtime はアバターとワールドオブジェクトを区別しない。

- 同一構造
- 同一判定
- 同一収益処理

---

## 15. パフォーマンス要件

- 差分更新を基本とする
- フル再評価を最小化
- 非同期処理を許容
- キャッシュ利用

---

## 16. diagnostics

Runtime 異常を記録する。

- 判定失敗
- 装着失敗
- 権利違反
- 収益処理失敗
