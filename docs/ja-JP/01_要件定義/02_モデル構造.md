# モデル構造

## 1. 目的

本仕様は、SansaVRM におけるモデルの内部構造を定義する。

SansaVRM は単一モデルではなく、
複数の構成要素（パーツ）を組み合わせて構成される
「構成可能なVRモデル基盤」として設計される。

---

## 2. 基本方針

- フォーマット非依存（glTF / VRM / URDF 等を統合）
- 人型非依存
- 単体モデルと構成モデルの両方を扱う
- モデル構造は拡張可能であること
- モデル構造は権利・許諾制御と連携する

### 2.1 本仕様とメタモデル仕様の関係

本仕様は、SansaVRM におけるモデル構造の概念および責務を定義するものである。

具体的なデータ構造、スロット定義、プロパティ構成、データ形式については
「メタモデル仕様」にて定義する。

本仕様では、それらの詳細構造には立ち入らず、
構造の役割および相互関係のみを扱う。

---

## 3. 全体構造

SansaVRM は以下の3層で構成する。

- Core Layer（共通核）
- Slot Layer（機能スロット）
- Format Extension Layer（形式別拡張）

本構造は論理構造であり、実際のデータ形式とは独立して定義される。

---

## 4. Core Layer

共通核として、モデルの基本構造および描画・変形・動作に必要な最小限の情報を保持する。

具体的な構成要素（ノード構造、形状、材質、アニメーション等）は
利用するフォーマットに依存しつつ統一的に扱われる。

詳細な構造はメタモデル仕様にて定義する。

---

## 5. Slot Layer

Slot Layer は、モデルの意味・機能を定義する拡張領域である。

### 5.1 Slot の概念

Slot は、モデルの機能・意味・制御情報を拡張的に保持する仕組みである。

Slot の種類は固定されず、用途に応じて拡張可能である。

代表的な例として以下がある（詳細はメタモデル仕様を参照）：

- Rig Slot
- Humanoid Mapping Slot
- Extended Mapping Slot
- Morph Slot
- Animation Slot
- Physics Slot
- Equipment Slot
- Compatibility Slot
- Semantic Tag Slot
- Rights Slot
- State Slot

---

## 6. 各 Slot 定義

### 6.1 Rig Slot

可動構造を統一的に表現する。

異なるフォーマットの構造定義（例：URDF等）を統合的に扱う。

---

### 6.2 Humanoid Mapping Slot

人型モデルへの標準的なマッピングを定義する。

具体的な部位定義はメタモデル仕様にて定義する。

---

### 6.3 Extended Mapping Slot

人型以外の構造を定義するための拡張マッピング領域である。

任意の構造を定義可能とする。

---

### 6.4 Morph Slot

形状変形および外観変化に関する情報を管理する。

ブレンドシェイプ相当の変形に加え、
材質変化およびテクスチャ変化を伴う表現を扱えるものとする。

具体的な構造はメタモデル仕様にて定義する。

---

### 6.5 Animation Slot

アニメーションに関する情報を管理する。

### 6.5.1 State Slot

State Slot は、モデルまたはモジュールの状態切替を定義する。

状態には、表示・非表示、装着・非装着、候補の切替、
表情切替、外観切替などを含む。

切替条件、状態識別子、状態間の対応関係の詳細は
メタモデル仕様にて定義する。

---

### 6.6 Physics Slot

物理挙動に関する情報を定義する。

---

### 6.7 Equipment Slot

モデル同士の接続・装着関係を定義する。

衣装・装備・小物などのモジュール構成を実現するために使用される。

具体的な構造およびプロパティはメタモデル仕様で定義する。

---

### 6.8 Compatibility Slot

構造的適合条件を定義する。

モデル間の組み合わせ可否を判定するために使用される。

具体的な条件定義はメタモデル仕様にて定義する。

---

### 6.9 Semantic Tag Slot

モデルの意味分類を定義する。

分類は用途やシステムに応じて拡張可能とする。

例（非限定）：

- humanoid
- quadruped
- robot
- creature
- world_object

---

### 6.10 Rights Slot

権利・利用許諾情報を保持する。

詳細は「[権利・許諾制御](./03_権利・許諾制御.md)」を参照。

---

## 7. モデル構成（Composable Model）

SansaVRM は複数モデルの組み合わせを前提とする。

### 7.1 構成単位

モデルは複数の構成単位から成り立つ。

構成単位の種類は固定されず、用途に応じて拡張可能である。

例（非限定）：

- Body
- Clothing
- Accessory
- Equipment
- Prop
- Module
- World Object

---

### 7.2 構成方法

- Equipment Slot により接続
- Compatibility Slot により適合判定
- Rights Slot により利用制御

---

## 8. モデル分類

SansaVRM は用途別に分類可能とする。

- Avatar Model
- Wearable Model
- Accessory Model
- Prop Model
- World Object Model
- Robot Model
- Creature Model
- Composite Model

---

## 9. マッピング方針

- VRM → Humanoid Mapping
- URDF → Rig Slot
- 非対応構造 → Extended Mapping
- 非可逆変換 → diagnostics に記録
- URDF 等の構造定義フォーマットは、Rig Slot を通じて統合される
- ヒューマノイド構造へのマッピングは必須ではなく、
  必要に応じて部分的または拡張的に適用される

---

## 10. 制約

- Slot は拡張可能であること
- Core Layer は最小限に保つ
- Slot に依存関係を持たせすぎない
- Slot の構造定義は本仕様では確定しない（メタモデル仕様で定義する）
- 特定フォーマット（VRM / glTF / URDF）に依存しないこと

---
