# Meta VRM メタモデル仕様 v0.1

## 1. 目的

Meta VRM は、VRM 0.x / 1.x を包含し、
URDF を含む任意構造モデルを扱うための内部共通表現を定義する。

## 2. 設計原則

- フォーマット非依存
- 人型非依存
- 非可逆変換の明示
- 拡張可能構造

## 3. 全体構造

Meta VRM は以下の3層で構成する。

- Core Layer
- Slot Layer
- Format Extension Layer

---

## 4. Core Layer

共通核として以下を保持する。

- Node Tree
- Transform
- Mesh
- Material
- Morph Target
- Animation
- Metadata Container

---

## 5. Slot Layer

### 5.1 Slot 一覧

- Rig Slot
- Humanoid Mapping Slot
- Extended Mapping Slot
- Morph Slot
- Animation Slot
- Physics Slot
- Attachment Slot
- Semantic Tag Slot
- Rights Slot

---

## 5.2 Rig Slot

- link / joint / bone 構造を統一表現

---

## 5.3 Humanoid Mapping Slot

- head, spine, arm など人型標準対応

---

## 5.4 Extended Mapping Slot

- tail, leg_01, wing 等
- 人型外構造

---

## 5.5 Morph Slot

- 表情
- 形状変形
- 機械ギミック変形

---

## 5.6 Animation Slot

- Animation Clip
- 状態定義
- 用途タグ

---

## 5.7 Physics Slot

- 制約
- バネ
- 衝突補助

---

## 5.8 Attachment Slot

- 装備
- 追加パーツ
- 接続点

---

## 5.9 Semantic Tag Slot

- humanoid
- quadruped
- arachnid
- robot

---

## 5.10 Rights Slot

権利・利用許諾情報を保持する。

構造:

- normalized
  - authors
  - license
  - commercial_use
  - redistribution
  - modification
- source_raw
  - 元形式情報
- diagnostics
  - 変換不能情報

---

## 6. Format Extension Layer

- VRM0 Extension
- VRM1 Extension
- URDF Extension
- 将来拡張

---

## 7. マッピング方針

- URDF は Rig Slot に変換
- 人型部分は Humanoid Mapping に投影
- 非対応構造は Extended Mapping に格納
- 非可逆変換は diagnostics に記録

---

## 8. 今後の拡張

- Khronos 拡張統合
- マルチアバター構造
- モジュール化アバター
