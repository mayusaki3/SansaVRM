# テスト結果レポート仕様

## 1. 目的

本ドキュメントは、SansaVRM におけるテスト結果の出力形式および集計方法を定義する。

本仕様の目的は以下とする。

- テスト結果の可視化
- テスト仕様（testspec）との対応関係の明確化
- CI における機械的検証の実現
- HLDocS Traceability との連携

---

## 2. 基本方針

テスト結果は以下の単位で管理する。

```md
・テストケース単位（test_id）
・テスト種別単位（カテゴリ）
・全体集計
```

Rust の `#[test]` は単なる実行単位とし、テスト仕様上の識別子は別途定義する。

---

## 3. テストID規則

### 3.1 形式

```text
<CATEGORY>-<TYPE>-<NNN>
```

例：

```text
VALIDATOR-ID-001
VALIDATOR-REF-002
CORE-MODEL-001
```

---

### 3.2 要素

| 要素 | 内容 |
|---|---|
| CATEGORY | 機能領域（VALIDATOR / CORE / CONVERT など） |
| TYPE | テスト種別（ID / REF / STATE / SCHEMA など） |
| NNN | 連番（001〜） |

---

### 3.3 Rustテストとの対応

Rust のテスト関数名は以下とする。

```rust
#[test]
fn validator_id_001_duplicate_module_id_should_fail() {
    ...
}
```

命名規則：

```md
・小文字 + スネークケース
・test_id を先頭に含める
```

---

## 4. テスト結果モデル

### 4.1 ステータス

```text
OK
NG
SKIP
```

---

### 4.2 出力単位

```json
{
  "test_id": "VALIDATOR-ID-001",
  "name": "module_id 重複検出",
  "status": "OK",
  "message": ""
}
```

---

## 5. レポート形式

---

### 5.1 Markdown形式

```md
# テスト結果

## 一覧

| テストID | 名称 | 結果 |
|---|---|---|
| VALIDATOR-ID-001 | module_id 重複検出 | OK |
| VALIDATOR-REF-001 | 未存在 module 参照 | OK |

## 集計

- 全テスト数：2
- 成功：2
- 失敗：0
- スキップ：0
```

---

### 5.2 JSON形式

```json
{
  "summary": {
    "total": 2,
    "success": 2,
    "failed": 0,
    "skipped": 0
  },
  "results": [
    {
      "test_id": "VALIDATOR-ID-001",
      "status": "OK"
    },
    {
      "test_id": "VALIDATOR-REF-001",
      "status": "OK"
    }
  ]
}
```

---

## 6. 生成方法

### 6.1 開発時

```md
・cargo test を実行
・テスト名から test_id を抽出
・手動またはスクリプトで集計
```

---

### 6.2 CI時

```md
・cargo test -- --format json を使用
・JSONを解析し test_id を抽出
・レポートを自動生成
```

---

## 7. トレーサビリティ連携

テスト結果は testspec の `sec_id` と紐づく。

```text
test_id → testspec(sec_id)
```

将来的に以下の関係を構築する。

```text
spec(sec_id)
  ↕
testspec(sec_id)
  ↕
test_id
  ↕
code (@hldocs.ref)
```

---

## 8. 制約

```md
・test_id は一意でなければならない
・test_id の変更は禁止（履歴維持）
・Rustテスト名と test_id は一致させる
```

---

## 9. 非スコープ

```md
・テスト実行フレームワークの実装
・CIツールの詳細設定
・レポートUI
```

---

## 10. 結論

SansaVRM では、テスト結果を test_id 単位で管理し、Markdown および JSON 形式で出力する。

これにより、テスト仕様・実装・CI を統合した品質管理を実現する。
