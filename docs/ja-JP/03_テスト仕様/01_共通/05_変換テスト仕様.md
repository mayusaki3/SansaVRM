# 変換テスト仕様

## 1. 目的

本仕様は、[変換仕様](../02_仕様/07_変換仕様.md) に基づく変換処理（Importer / Exporter）の検証テスト仕様を定義する。

本仕様の目的は以下とする。

- 外部フォーマットと SansaVRM Core 間の変換が正しく行われることを確認する
- 構造・接続・状態・属性が正しくマッピングされることを検証する
- 非可逆変換が適切に扱われることを保証する
- `source_raw` および `diagnostics` の動作を検証する

---

## 2. テスト対象

以下の変換処理を対象とする。

- glTF → SansaVRM（Importer）
- VRM → SansaVRM（Importer）
- URDF → SansaVRM（Importer）
- SansaVRM → glTF（Exporter）
- SansaVRM → VRM（Exporter）
- SansaVRM → URDF（Exporter）

---

## 3. テスト分類

---

### 3.1 Import（外部 → SansaVRM）

対象：

- glTF / VRM / URDF

検証内容：

- Model 構造が正しく生成されること
- Module / Slot / State が正しくマッピングされること
- ID が一意かつ安定して生成されること
- 接続（connections）が正しく構築されること
- property_type が正しく設定されること

---

### 3.2 Export（SansaVRM → 外部）

対象：

- glTF / VRM / URDF

検証内容：

- 外部フォーマットとして正しい構造が生成されること
- スキーマ（glTF / VRM / URDF）に準拠していること
- 必須要素が欠落していないこと
- extension が正しく出力されること
- property_type に基づき geom / actuator / sensor が生成されること
- property_type 未設定時のフォールバックテスト
- VRM 0.x / 1.0 の差異が正しく反映されること

---

### 3.3 マッピング検証

対象：

- Module ↔ node / link
- Slot ↔ joint / node関係
- State ↔ animation / expression
- Property分類（property_type → geom / actuator / sensor）
- joint Connection のみが MuJoCo に変換されること

検証内容：

- 双方向で整合すること
- 不整合が発生しないこと

---

### 3.4 ID安定性検証

検証内容：

- 同一入力から同一IDが生成されること
- 複数回実行しても変化しないこと

---

### 3.5 非可逆変換

検証内容：

- 失われる情報が `source_raw` に保存されること
- `diagnostics` に warning が記録されること
- error にならないこと（仕様に従う）

---

### 3.6 diagnostics 検証

対象：

- type
- severity
- message

検証内容：

- 変換時の問題が正しく記録されること

---

### 3.7 Round-trip前提検証

検証内容：

- Import → Export の連続処理が成立すること
- Export → Import の連続処理が成立すること
- 中間状態（SansaVRM）が整合性を維持すること

※詳細な往復一致検証は「06_ラウンドトリップテスト仕様.md」で定義する

---

## 4. テスト方針

---

### 4.1 正常系

- 正しい外部データが正常に変換されること
- 出力がフォーマット仕様を満たすこと

---

### 4.2 異常系

- 不正データが適切に検出されること
    - 必須要素欠落
    - 型不一致
    - 不正構造
- 変換不能部分が diagnostics に記録されること

---

### 4.3 網羅性

以下を必ずカバーする。

- 全フォーマット（glTF / VRM / URDF）
- Import / Export 双方向
- 構造 / 接続 / 状態

---

### 4.4 再現性

- 同一入力で同一出力となること

---

## 5. テストケース設計

各テストケースは以下を持つ。

- テストID
- テスト分類
- 入力フォーマット
- 入力データ
- 変換処理
- 期待結果
- diagnostics

---

---

### CONVERT-GLTF-001

- テストID：CONVERT-GLTF-001
- 分類：Import
- 内容：glTF → SansaVRM
- 期待結果：Model が生成される

---

### CONVERT-VRM-002

- テストID：CONVERT-VRM-002
- 分類：Import
- 内容：VRM → SansaVRM
- 期待結果：State / Property が正しく生成される

---

### CONVERT-URDF-003

- テストID：CONVERT-URDF-003
- 分類：Import
- 内容：URDF → SansaVRM
- 期待結果：Module / Slot / Connection が生成される

---

### CONVERT-GLTF-004

- テストID：CONVERT-GLTF-004
- 分類：Export
- 内容：SansaVRM → glTF
- 期待結果：glTF構造が正しく出力される

---

### CONVERT-VRM0-005

- テストID：CONVERT-VRM0-005
- 分類：Export
- 内容：SansaVRM → VRM 0.x
- 期待結果：
  - VRM 0.x 仕様に準拠した出力が生成される
  - VRM 1.0 専用要素は出力されない
  - 非対応要素は diagnostics に warning として記録される

---

### CONVERT-VRM1-006

- テストID：CONVERT-VRM1-006
- 分類：Export
- 内容：SansaVRM → VRM 1.0
- 期待結果：
  - VRM 1.0 仕様に準拠した出力が生成される
  - 拡張要素（expression / constraint 等）が適切に出力される

---

## 7. 成功条件

以下を満たすこと。

- 変換結果が仕様と一致する
- ID が安定している
- diagnostics が期待と一致する
- source_raw が適切に保持される

---

## 8. 失敗条件

以下の場合は失敗とする。

- 変換結果が不正
- 構造が欠落
- ID が不安定
- diagnostics が不正

---
